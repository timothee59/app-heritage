# Story 7.1 - Améliorer l'accessibilité pour les seniors

## Description
En tant qu'utilisateur senior (65+), je veux que les boutons, icônes et textes soient plus grands afin de pouvoir utiliser l'application confortablement sans effort visuel.

## Contexte
Les tests avec des utilisateurs réels ont révélé que certains éléments sont trop petits pour une utilisation confortable par des personnes âgées :
- Boutons de navigation trop petits
- Bouton X pour fermer le zoom photo difficile à toucher
- Textes trop petits dans certaines zones

## Critères d'acceptation

### Boutons et zones tactiles
- [ ] Tous les boutons de navigation ont une taille minimale de 44x44px
- [ ] Le bouton X de fermeture du zoom photo est agrandi (minimum 48x48px)
- [ ] Les boutons de filtres sont plus grands et plus espacés
- [ ] Les boutons d'action (ajouter photo, supprimer, restaurer) sont agrandis

### Typographie
- [ ] Les titres de section sont agrandis d'un niveau (text-lg → text-xl)
- [ ] Les libellés des boutons utilisent text-base minimum
- [ ] Les textes descriptifs restent lisibles (minimum 16px)
- [ ] Le contraste texte/fond reste conforme AA

### Navigation
- [ ] Les icônes de navigation (flèches, menu) sont agrandies (w-6 h-6 minimum)
- [ ] L'espacement entre les éléments cliquables est suffisant pour éviter les erreurs de tap

### Responsive
- [ ] L'agrandissement n'entraîne pas de débordement sur iPhone/iPad
- [ ] La grille de la galerie reste cohérente (2 colonnes mobile, 3-4 desktop)

## Éléments à modifier

### gallery.tsx
- Boutons de filtres (Tous, Je le veux!, etc.)
- Bouton d'ajout d'objet
- Menu utilisateur
- Toggle "Afficher supprimées"

### item-detail.tsx
- Bouton retour
- Boutons de navigation photo (précédent/suivant)
- Bouton X fermeture zoom
- Boutons de préférence
- Bouton supprimer/restaurer

### Composants modaux
- Boutons de confirmation dans les AlertDialog
- Boutons dans les Dialog

## Notes techniques
- Utiliser les variants Shadcn appropriés (size="lg")
- Augmenter les icônes Lucide (w-6 h-6 ou plus)
- Vérifier avec Lighthouse accessibility audit
- Tester manuellement sur iPad/iPhone

## Priorité
Haute - Bloquant pour l'adoption par les utilisateurs cibles
