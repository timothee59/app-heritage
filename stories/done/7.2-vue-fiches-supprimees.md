# Story 7.2 - Vue dédiée pour les fiches supprimées

## Description
En tant qu'utilisateur, je veux que les fiches supprimées soient regroupées dans une vue séparée afin de ne pas polluer la galerie principale avec des éléments grisés.

## Contexte
Actuellement, les fiches supprimées apparaissent en fin de liste dans la galerie principale (grisées et barrées). Cela peut créer de la confusion et encombrer visuellement l'interface, surtout si plusieurs fiches ont été supprimées.

## Critères d'acceptation

### Galerie principale
- [ ] La galerie principale n'affiche plus les fiches supprimées
- [ ] Le toggle "Afficher les fiches supprimées" est remplacé par un accès à la vue dédiée
- [ ] Un indicateur montre le nombre de fiches supprimées (badge ou compteur)

### Vue des fiches supprimées
- [ ] Une nouvelle vue/page dédiée affiche uniquement les fiches supprimées
- [ ] Les fiches sont triées par date de suppression (plus récentes en premier)
- [ ] Chaque fiche affiche :
  - La photo principale (grisée)
  - Le titre (barré)
  - Qui a supprimé et quand
  - Un bouton "Restaurer" bien visible

### Navigation
- [ ] Accès facile depuis la galerie (bouton ou onglet "Corbeille")
- [ ] Bouton retour pour revenir à la galerie
- [ ] L'icône Trash2 identifie clairement cette section

### Restauration
- [ ] Le bouton Restaurer fonctionne depuis cette vue
- [ ] Après restauration, la fiche disparaît de la liste
- [ ] Un message de confirmation s'affiche
- [ ] La fiche restaurée réapparaît dans la galerie principale

### État vide
- [ ] Message explicite si aucune fiche supprimée ("Aucune fiche dans la corbeille")
- [ ] Suggestion de retourner à la galerie

## Approche technique

### Option A : Nouvelle page /deleted
- Créer une page `deleted-items.tsx`
- Ajouter la route dans App.tsx
- Bouton d'accès dans la barre de filtres de la galerie

### Option B : Filtre dédié
- Ajouter un filtre "Corbeille" dans les filtres existants
- Modifier l'affichage quand ce filtre est actif
- Moins de navigation, plus intégré

### API
- Utiliser le paramètre existant `showDeleted=true`
- Ou créer un endpoint dédié `/api/items/deleted`

## Notes UX pour seniors
- Bouton Restaurer bien visible (grande taille, couleur contrastée)
- Libellés explicites ("Corbeille", "Restaurer la fiche")
- Éviter la surcharge d'informations
- Transitions simples entre les vues

## Priorité
Moyenne - Amélioration de l'expérience utilisateur
