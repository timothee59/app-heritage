# Story 5.8 - Photo responsive avec zoom

## Objectif
Améliorer l'affichage des photos dans la page de détail d'un bien pour qu'elles soient proportionnelles à l'écran et permettre un zoom.

## Problème actuel
Les photos prennent trop de place sur la page de détail, aussi bien sur mobile que sur PC, ce qui rend la navigation moins agréable.

## Solution proposée

### 1. Taille responsive des photos
- **Mobile (< 640px)** : La photo occupe maximum 50% de la hauteur de l'écran
- **Tablette (640px - 1024px)** : La photo occupe maximum 40% de la hauteur de l'écran  
- **Desktop (> 1024px)** : La photo occupe maximum 35% de la hauteur de l'écran
- Conserver le ratio d'aspect original de la photo (pas de déformation)

### 2. Fonctionnalité de zoom
- Au tap/clic sur la photo, ouvrir une vue plein écran (lightbox)
- Dans la lightbox :
  - Photo affichée en grand, centrée
  - Possibilité de zoomer avec pinch (mobile) ou molette (desktop)
  - Bouton X pour fermer
  - Fond sombre semi-transparent
- Navigation entre photos possible dans la lightbox (si plusieurs photos)

## Critères d'acceptation
- [ ] Les photos sont dimensionnées de manière proportionnelle selon le device
- [ ] Un indicateur visuel montre que la photo est cliquable (ex: icône loupe)
- [ ] Le tap/clic ouvre une lightbox avec la photo en grand
- [ ] Le zoom fonctionne sur mobile (pinch) et desktop (molette)
- [ ] La lightbox peut être fermée facilement
- [ ] Le carousel reste fonctionnel pour naviguer entre les photos

## Notes techniques
- Utiliser CSS `max-height` avec `vh` pour la hauteur responsive
- Utiliser un composant Dialog/Modal pour la lightbox
- Considérer une librairie légère pour le zoom (ou CSS transform)
